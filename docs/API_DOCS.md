# API Reference

Full interactive docs are auto-generated by FastAPI at `/docs` (Swagger UI)
and `/api/v1/openapi.json` once the backend is running. This file is a
human-readable summary for the report.

Base URL (local dev): `http://localhost:8000/api/v1`

## Vehicles

| Method | Path | Purpose | Phase |
|---|---|---|---|
| GET | `/vehicles/search?make=&model=` | Autocomplete suggestions (UX convenience only, never a hard restriction) | 1 |
| GET | `/vehicles/lookup?make=&model=&year=` | Accepts ANY free-text input. Always returns `verified` or `estimated` — never a rejection | 1 |
| POST | `/vehicles/submit-unlisted` | Open submission queue, no cap, no auth required in Phase 1 | 1 |
| POST | `/vehicles/submissions/{id}/approve` | Admin promotes a submission to a verified spec (auth added Phase 6) | 1 |

## Predictions

| Method | Path | Purpose | Phase |
|---|---|---|---|
| POST | `/predict/risk-score` | Component wear risk + confidence interval. Phase 1 = rule-based placeholder; Phase 2 = trained model + SHAP | 1 (placeholder) / 2 (real) |

## Response tagging convention

Every response that includes vehicle-spec data includes a `match_type` field
(`verified` \| `estimated`). Every prediction response includes
`confidence_interval_low` / `confidence_interval_high` and a `disclaimer`
string. Every aggregated report response omits any region below the
minimum-report-count privacy threshold.
